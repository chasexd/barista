Development Environment Setup

Follow these steps on your Macintosh to load a development instance of the system:

  1. Install Docker Desktop
  2. Install Brew
  3. Install NVM using brew install nvm and don't forget to update your .bash_profile
  4. Install Node 10.x LTS using NVM: nvm install v10.16.0 and ensure that it is the version in use by checking ith the node -v command.
  5. Install the Yarn dependency manager using the npm i -g yarn command.
  6. Clone the repository
  7. From within each application subdirectory (barista-api, barista-scan, barista-web) load dependencies using the yarn install command.
  8. In a console window from the root of the barista-api/ folder type yarn compose:dev up (Make sure that the Docker daemon is started)
  9. From within the barista-api directory issue the yarn db:reset command to configure the database.
  10. From within each application subdirectory (barista-api, barista-scan, barista-web) in separate console windows start each application tier using the yarn start:dev command.
  11. Once the application is started the following instances should be available from your localhost:
    • Barista Web: The system's main web user UI.
    • Barista Api: The REST interface Swagger documentation.
    • Barista Scan (No interface), pulls it's work from the Bull/Redis work queue.
    • PostgreSQL Development database running at: postgresql://postgres:password@db:5432/barista-dev
    • PostgreSQL Test database running at: postgresql://postgres:password@db:5433/barista-test. The test database is used by the end-to-end (e2e) tests.
    • The following tools are provided as a convenience for development and are not required to run the system:
      • PostgresAdmin: A web based administrative tool for the Postgres database [postgres@admin.com:password].
      • Arena Queue UI: An interactive dashboard for the Bull queue product. See: https://github.com/bee-queue/arena
      • Redis Commander: A GUI Redis interface.
      • Dozzle: A GUI to examine the log output from the various services. See: https://dozzle.dev/

#Notes

  • The barista-api/docker/*.yml Docker Compose configuration files provide useful information about the configuration of the dev environment including login credentials for configured services.
  • To stop the back end services you can issues the command: yarn compose:dev down

#Dependencies

In order to execute project scans, the following software packages must be installed:

  • OpenJDK 8 (OWASP Dependency Check)
    • brew tap adoptopenjdk/openjdk
    • brew cask install adoptopenjdk8
  • Maven (Java project dependencies and licenses)
    • brew install maven
  • Nuget (.NET project dependencies and licenses)
    • brew install nuget
